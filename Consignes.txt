Projet MVC
Gestion des articles
Un article doit avoir une image de couverture (format de l'image libre). Si l'image est absente, prévoir l'affichage d'une image par défaut.
Export
Créer 3 scripts d'export CSV pour les utilisateurs, articles, et commentaires. Format d'export :
Utilisateurs :
id, nom, email, password, actif, admin, nb articles, nb commentaires
Articles :
id, titre, contenu, date (format Y-m-d H:i:s), id_auteur, nb commentaires
Commentaires :
id, titre, contenu, date (format Y-m-d H:i:s), id_auteur, id_article

Front:
Accueil
Affichage des 10 derniers articles (titre, date, auteur et image de couverture) avec pagination pour voir les articles plus anciens
Page d'un article
Afficher le détail (auteur (nom, image 150*150), image de couverture, titre, date, contenu) et les commentaires sous l'article.
Sous la liste de commentaires : afficher le formulaire pour créer un nouveau commentaire. (un utilisateur doit être connecté pour laisser un commentaire)
À l'enregistrement du commentaire, l'auteur doit être averti par e-mail (en utilisant le design pattern observer)
Sur le nom de l'auteur, mettre un lien vers son profil.
Auteurs
Faire une page avec la liste des auteurs (image de profil (150*150), nom, nombre d'articles posté, nombre de commentaires posté). Mettre un lien sur le nom pour voir le profil de l'auteur.
Profil : 
Afficher l'image de profil (500*500), le nom, l'email (pas le mot de passe ni le statut actif ou non). 
Lister les articles de l'auteur (image de couverture, titre (avec lien vers la page d'un article), date, nombres de commentaires)
Lister les articles sur lesquels l'auteur à laisser au moins un commentaire.

Format des photos de profil utilisateurs : 800*800, 500*500, 150*150, si l'image est absente, afficher une image par défaut au bon format

Faites attention :
- aux commentaires dans le code
- au placement logique de vos fonctions & méthodes dans les bons controller ou model.
- à l'indentation

POINTS BONUS :
- interface graphique agréable
- export des articles et commentaires par intervalle de date
- ajouter un compteur de visites sur les articles (quand on affiche le détail d'un article) et le profil d'un utilisateur. Afficher ces compteurs dans l'admin.
- ajouter une fonction de partage par e-mail d'un article
